## How to Use This Log

- **Version numbering:** Increment the patch version for minor changes (1.0 → 1.1). Use a minor increment for significant scope changes (1.0 → 2.0).
- **Date:** The date the change was approved, not when it was written.
- **Author:** Who made the change to the spec.
- **Change:** A concise description of what changed and in which document.
- **Requested By:** Who requested this change (customer, PM, developer).
